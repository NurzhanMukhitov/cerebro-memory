# Комплексный аудит бота Cerebro (февраль 2026)

Проверка всех блоков после серии изменений (SOUL/AGENTS, погода, время, память, фото). Что было сломано, что исправлено, что проверить.

---

## 1. Время («который час?»)

| Проблема | Причина | Решение | Статус |
|----------|---------|---------|--------|
| Время на час назад | Сервер в UTC, gateway без TZ → `session_status` возвращал UTC | В unit `openclaw-gateway.service` добавлено `Environment=TZ=Europe/Madrid` | ✅ Исправлено на VPS |

**Проверка:** написать «Который час?» — время должно совпадать с меткой сообщения в Telegram (Барселона).

## 2. Календарь (расписание, встречи)

| Проблема | Причина | Решение | Статус |
|----------|---------|---------|--------|
| «Нет доступа» / «календарь не подключён» | Модель не вызывала tool, отвечала сама | В SOUL и TOOLS: «сначала вызови инструмент caldav/khal/calendar» | ✅ Правило в репо |
| События на час назад | khal без local_timezone | В `~/.config/khal/config` секция `[locale]` → `local_timezone = Europe/Madrid` | ✅ По README |

**Проверка:** «Что на неделе?», «Встречи на завтра» — бот должен вызвать календарный tool и ответить по факту.

## 8. Скорость ответа и rate limit

| Проблема | Причина | Решение | Статус |
|----------|---------|---------|--------|
| Долгий ответ, «rate limit» | Большой контекст (дубль BOOTSTRAP, большой AGENTS), лимиты OpenAI TPM/RPM | BOOTSTRAP минимальный; AGENTS сокращён до ~10k символов; при лимите — не слать подряд запросы | ✅ Сделано |
| Compaction падает по TPM | После ответа OpenClaw делает суммаризацию тем же API | Не ставить compaction.mode = "off" (падение gateway); ждать сброса TPM 1–2 мин | ✅ Задокументировано |

**Проверка:** одиночный запрос без очереди — обычно 8–25 с. При «rate limit» в логах — подождать минуту.

## 9. Чек-лист проверки на VPS после деплоя

После `git pull` в `~/cerebro-memory` и `systemctl --user restart openclaw-gateway`:

1. **Время:** «Который час?» → время Барселоны, совпадает с меткой в Telegram.
2. **Профиль:** «Что ты знаешь обо мне?» → ответ из профиля, без «нет доступа».
3. **Погода:** «Погода на завтра в Барселоне» → вызов weather, есть прогноз.
4. **Календарь:** «Что на неделе?» → вызов календаря, список событий или «нет встреч».
5. **Фото:** отправить фото жены + «Кто на фото?» → «похоже, это твоя жена» (из USER).
6. **Напоминание:** «Напомни через 2 минуты тест» → через 2 мин приходит напоминание.

## 10. Файлы и симлинки (напоминание)

| Файл в workspace | Репо | Назначение |
|------------------|------|------------|
| SOUL.md | core/manifest.md | Персона, время, календарь, погода, профиль, фото. |
| USER.md | core/user-profile.md | Профиль, близкие (жена на фото), тон. |
| AGENTS.md | core/agents.md | CoS, память, советники, DF. |
| TOOLS.md | core/tools.md | Когда вызывать session_status, календарь, weather, профиль/фото. |
| BOOTSTRAP.md | **не из репо** | На VPS — минимальный (1 строка), не симлинк на manifest. |

Память `workspace/memory/*.md` может не входить в bootstrap; важные факты (кто на фото) дублировать в USER.
