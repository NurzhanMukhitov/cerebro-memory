# Подсказки по инструментам (TOOLS)

Краткая шпаргалка: когда какой tool вызывать. Полные правила — в SOUL и AGENTS.

## Время и дата

- **`session_status`** — единственный источник текущего времени. Вызывать при любом вопросе про время, дату, планирование, расписание, календарь, напоминания. В ответе смотреть строку `🕒 Time:` (локальное время Europe/Madrid). Не кешировать и не повторять время из прошлого ответа.

## Календарь

- **caldav-calendar** (skill): инструменты с именами/описаниями, содержащими `caldav_calendar`, `khal`, `calendar`. Для вопросов «что на неделе?», «встречи на завтра», «расписание на понедельник» — вызвать tool списка событий с нужным диапазоном дат (например `khal list today 7d`). Сначала вызов — потом ответ. Не писать «нет доступа» и не просить скрин, пока не вызван tool.
- **При добавлении события в календарь:** передавать время в **Europe/Madrid** (не UTC), иначе в календаре будет сдвиг на час. Если пользователь сказал время «по Москве» / «Мск» — сначала конвертировать MSK → Madrid, затем добавить уже время по Мадриду. Личные события без указания пояса = время по Мадриду.
- **После любого изменения календаря (khal add/delete/edit):** вызвать **`vdirsyncer sync`** (exec на VPS), иначе изменения не попадут на CalDAV‑сервер и на Mac останутся дубликаты или старые события.

## Профиль пользователя и фото
- Вопросы «что ты знаешь обо мне?», «покажи мой профиль»: данные уже в контексте (USER.md в bootstrap). Не вызывать read_file; отвечать из контекста.
- «Кто на фото?»: если в USER или в памяти есть факт о человеке (например жена пользователя на фото — запомнено) — отвечать из контекста. Не отвечать «не могу определить», если идентификация уже есть в USER/памяти.

## Погода

- При любом вопросе про погоду (сейчас, на завтра, по городу, для велосипеда и т.п.) — **сначала вызвать tool weather** (по имени/описанию с `weather`). Не отвечать «нет доступа» без вызова. Локация из профиля (Барселона), либо из запроса пользователя.

## Новости / RSS (rss-reader)

- Вопросы «какие новости?», «что в лентах?», «новости» — **сначала вызвать tool** из skill rss-reader (по имени/описанию с `rss` или `feed`). Ленты настраиваются на VPS в `~/.openclaw/workspace/skills/rss-reader/data/feeds.json`; добавление: `node scripts/rss.js add "URL" --category имя` из каталога skill.

## Strava (тренировки)

- Интеграция Strava уже настроена (STRAVA_ACCESS_TOKEN в окружении). Вопросы про последнюю тренировку, активности, километраж — **вызывать exec**: curl к Strava API (`https://www.strava.com/api/v3/athlete/activities?per_page=10`, для деталей — `/activities/ACTIVITY_ID`), заголовок `Authorization: Bearer $STRAVA_ACCESS_TOKEN`. Документация: skill strava (SKILL.md). Сначала exec с запросом к API — потом ответ. Не писать «нужно подключить» — подключено.
- **Для советников Sport / Health / питание:** при рекомендациях по нагрузке, восстановлению, плану тренировок — использовать данные Strava. Если в workspace есть `data/strava-snapshot.md` (снимок активностей и статистики) — прочитать его через read; при необходимости актуализировать через exec к API или подождать следующего обновления снимка по cron.
- **Ultrahuman Ring (сон, Recovery, HRV):** если в workspace есть `data/ultrahuman-snapshot.md` (или `.txt`) — прочитать через read и учесть при рекомендациях по сну, восстановлению, нагрузке. Файл обновляется пользователем вручную (в приложении API нет для личного использования).
- **Apple Health (HealthKit):** при вопросе про данные Health, «весь период», «сколько данных» — **читать оба файла**: `data/apple-health-snapshot.md` и `data/apple-health-baseline.md`. Отвечать по объёму и содержимому обоих. Учитывать при рекомендациях Sport/Health.

## Google / Gog (gog)

- Вопросы про Gmail (письма, поиск, последние письма), Google Calendar, Drive — вызывать tool из skill **gog** (по имени/описанию с `gog`, `gmail`, `google`). Сначала вызов — потом ответ. Не писать «нет доступа» без вызова.
- **Подтверждение:** перед отправкой письма или созданием/изменением события в Google — кратко описать действие и запросить подтверждение пользователя. Не выполнять отправку писем и создание событий без явного «да»/«подтверждаю». Деструктивные действия — только после подтверждения.

## Остальные skills
- **browser / Headless Chrome** — открыть URL, прочитать страницу.
- **summarize** — краткий пересказ по ссылке.
- **openai-whisper-api** — транскрипция голоса (подставляется автоматически при голосовых в Telegram).
- **remind** — напоминания по времени и расписанию (фразы вида «напомни в 18:00», «напомни завтра в 10:00»).

Доступные tools видны в списке при запросе; вызывать только зарегистрированные, по их схеме параметров.
