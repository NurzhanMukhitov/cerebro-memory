# State Model

State Model описывает текущее состояние пользователя.

Состояние влияет на поведение всех агентов системы.

Каждый агент должен учитывать текущее состояние перед формированием рекомендаций.

---

# Источник состояния

Текущее состояние хранится в файле:

`state/current-state.json`

Этот файл содержит операционные параметры состояния пользователя.

---

# Пример структуры состояния

```json
{
    "date": "2026-03-05",
    "health_status": "OK",
    "energy_level": "normal",
    "work_mode": "NORMAL",
    "sleep_quality": "normal"
}
```

---

# Основные состояния системы

Система использует несколько типов состояния.

| State | Значение |
|---|---|
| NORMAL | обычный режим |
| RECOVERY | восстановление |
| SICK | болезнь |
| OVERREACHED | перегруз |
| HIGH_LOAD | высокая рабочая нагрузка |
| FOCUS_WORK | режим концентрации |

---

# Поведение доменов в зависимости от состояния

Каждый домен реагирует на состояние пользователя.

## Health

| State | Поведение |
|---|---|
| NORMAL | обычный режим |
| RECOVERY | рекомендации по восстановлению |
| SICK | избегать нагрузок |
| OVERREACHED | разгрузка и сон |
| HIGH_LOAD | поддержка режима восстановления |
| FOCUS_WORK | контроль стресса |

## Sport

| State | Поведение |
|---|---|
| NORMAL | обычный план тренировок |
| RECOVERY | лёгкие тренировки |
| SICK | отмена тренировок |
| OVERREACHED | разгрузочные дни |
| HIGH_LOAD | снижение объёма |
| FOCUS_WORK | короткие тренировки |

## Work

| State | Поведение |
|---|---|
| NORMAL | обычная рабочая нагрузка |
| HIGH_LOAD | уменьшить дополнительные задачи |
| FOCUS_WORK | приоритет рабочим задачам |
| RECOVERY | снизить интенсивность работы |
| OVERREACHED | минимизировать новые задачи |
| SICK | только критические задачи |

## Food

| State | Поведение |
|---|---|
| NORMAL | обычное питание |
| RECOVERY | питание для восстановления |
| SICK | лёгкая пища |
| OVERREACHED | восстановление энергии |
| HIGH_LOAD | поддержка энергии |
| FOCUS_WORK | простая и быстрая еда |

Telegram topic: **Nutrition**

## Home

| State | Поведение |
|---|---|
| NORMAL | обычные бытовые задачи |
| HIGH_LOAD | сокращение домашних задач |
| FOCUS_WORK | минимизация отвлекающих задач |
| RECOVERY | спокойный режим |
| OVERREACHED | перенос домашних задач |
| SICK | только необходимые задачи |

## Learning

| State | Поведение |
|---|---|
| NORMAL | обычный план обучения |
| HIGH_LOAD | уменьшение нагрузки |
| FOCUS_WORK | обучение по работе |
| RECOVERY | лёгкое обучение |
| OVERREACHED | пауза в обучении |
| SICK | пауза |

---

# Обновление состояния

Состояние может обновляться:

- Health Advisor
- Chief of Staff
- Work Advisor
- Sport Advisor

Но запись выполняется только через **Write Intent и Commit Layer**.

---

# Итог

State Model позволяет системе:

- учитывать текущее состояние пользователя
- адаптировать рекомендации агентов
- предотвращать перегруз
- синхронизировать поведение доменов
