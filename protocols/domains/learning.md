# Чат / топик «Learning»

## Назначение домена

Learning Domain отвечает за обучение пользователя, развитие навыков и освоение новых знаний.

Домен охватывает: изучение новых тем, развитие профессиональных навыков, образовательные планы, курсы и обучающие материалы, исследование новых областей знаний, структурирование учебных целей.

## Агент домена

**Primary Agent:** Learning Advisor

Роль: помогать структурировать обучение, формировать учебные цели, планировать образовательную деятельность, систематизировать знания, анализировать образовательные материалы, поддерживать прогресс обучения.

**Secondary:** Chief of Staff (согласование с другими задачами, планирование недели, баланс с работой и отдыхом).

## Маршрутизация

**Topic:** Learning | **Domain:** learning | **Primary Agent:** Learning Advisor

Типичные сообщения: вопросы по обучению, изучение новых тем, обсуждение образовательных материалов, планирование обучения.

## Основные файлы данных

- learning/goals.md — образовательные цели
- learning/plan.md — текущий план обучения

## Context Pack

**Mandatory:** learning/goals.md, learning/plan.md  
**Relevant:** general/weekly-plan.md, calendar/today, calendar/next-7-days  
**Optional:** work/projects.md, tech/notes.md

## Поведение при различных состояниях

Источник: state/current-state.json

- **NORMAL:** планирование обучения, изучение новых тем.
- **HIGH_LOAD:** лёгкие форматы обучения, избегать сложных образовательных задач.
- **FOCUS_WORK:** минимизировать образовательную активность, откладывать новые темы.
- **RECOVERY:** лёгкое обучение, чтение или просмотр материалов.

## Write Intent

Learning Advisor не записывает файлы напрямую. Изменения — через Write Intent → Commit Layer.

## Ограничения

Learning Advisor не принимает медицинские и финансовые решения, не изменяет файлы других доменов без Write Intent.

## Итог

Learning Domain отвечает за образовательное развитие. При конфликте приоритетов применяется Advisor Priority Model.
