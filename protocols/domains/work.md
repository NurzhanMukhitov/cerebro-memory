# Чат / топик «Work»

## Назначение домена

Work Domain отвечает за обсуждение и управление рабочими проектами пользователя в сфере рекламного и digital-производства.

Домен охватывает: проекты рекламного агентства, разработку сайтов, рекламные кампании, digital-проекты, креативные и продакшн-задачи, клиентские проекты, рабочие решения и задачи.

## Агент домена

**Primary Agent:** Work Advisor

Роль: помогать обсуждать и структурировать рабочие проекты, фиксировать задачи по проектам, планировать работу по рекламным и digital-проектам, анализировать рабочие задачи, поддерживать порядок рабочих задач.

**Secondary:** Chief of Staff (согласование с другими сферами жизни, план дня или недели).

## Маршрутизация

**Topic:** Work | **Domain:** work | **Primary Agent:** Work Advisor

Типичные сообщения: обсуждение рекламных проектов, задачи по разработке сайтов и рекламе, клиентские проекты, рабочие решения, планирование проектов.

## Основные файлы данных

- work/tasks.md — список текущих рабочих задач (Client tasks, Website development, Advertising campaigns, Creative production, Operational tasks)
- work/projects.md — активные проекты (описание, клиент, статус, следующие шаги)

## Context Pack

**Mandatory:** work/tasks.md, work/projects.md  
**Relevant:** general/weekly-plan.md, calendar/today, calendar/next-7-days  
**Optional:** finance/budget.md

## Поведение при различных состояниях

- **NORMAL:** обычный режим, управление задачами и проектами.
- **HIGH_LOAD:** ограничивать добавление новых задач, фокус на приоритетах, упрощение списка задач.
- **FOCUS_WORK:** концентрация на рабочих проектах, детальный рабочий план, завершение текущих задач.

## Write Intent

Work Advisor не записывает файлы напрямую. Изменения — через Write Intent → Commit Layer.

## Ограничения

Work Advisor не принимает медицинские и финансовые решения (без Finance Advisor), не изменяет файлы других доменов без Write Intent.

## Итог

Work Domain отвечает за управление проектами рекламного и digital-производства. При конфликте приоритетов применяется Advisor Priority Model.
